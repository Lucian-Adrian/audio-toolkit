## Phase 6: Plugin System

**Goal:** Enable third-party plugins to extend Audio Toolkit.

**Requirements Addressed:** PLUGIN-001 to PLUGIN-007, OPT-004

### Atomic Tasks

#### 6.1 Plugin Manager

- [x] **6.1.1** Create `src/orchestration/plugin_manager.py`
  - `PluginManager` class with class-level registries
  - `discover()` - Load built-in and third-party processors via entry_points
  - `_register_builtins()` - Register built-in processors (FixedSplitter, FormatConverter)
  - `_discover_entry_points()` - Discover third-party plugins via entry_points
  - `_load_plugin()` - Load single plugin with validation and error handling
  - `_register_processor()` - Register processor with interface validation
  - `register()` - Manual registration (can be used as decorator)
  - `get()` - Get processor instance by name
  - `get_class()` - Get processor class by name
  - `list_all()` - Return all registered processor instances
  - `list_by_category()` - Filter processors by category
  - `list_names()` - Return sorted list of processor names
  - `disable()` / `enable()` - Enable/disable plugins
  - `is_disabled()` / `get_disabled()` - Check disabled status
  - `reset()` - Reset to uninitialized state (for testing)
  - Module-level convenience functions: `discover()`, `get_processor()`, `list_processors()`
